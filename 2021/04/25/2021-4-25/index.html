hexo搭建博客的注意点

发表于 2021-04-25 更新于 2021-05-07 分类于 知识笔记 阅读次数：

本文讲述了笔者利用hexo框架搭建个人博客网站时候需要注意的点

初始搭建步骤

参考知乎枫叶大佬的详细教程：

https://zhuanlan.zhihu.com/p/102592286

枫叶大佬的搭建教程非常详细，适合小白入门，不过由于版本的问题，过去教程中的某些部分对于现在不再适用，笔者将在下文进行补充。

关于Post内图片上传的问题

方法一：

采用next主题默认的图片存放位置，将要加入post的图片放入到next主题下source/images中去，然后在md文件中引用的时候采用/images/图片名.后缀名的方式。这种方法较为简单，但是存在图片管理不方便的问题。

方法二：

将站点配置文件中的post_asset_folder选项设置为true，然后安装插件hexo-asset-image。这种方法需要注意的点是这个插件的版本问题。npm install hexo-asset-image --save这个命令安装的插件版本是1.0.0版本，而npm install https://github.com/CodeFalling/hexo-asset-image --save这个命令安装的是0.0.5版本，这两个版本对于图片文件的链接引用是不一样的。我们要做的是将某个post要用到的图片直接放入同名的文件夹中，然后md文件中采用文件夹名/文件名.后缀名格式进行引用，因此需要选择的是0.0.5版本的插件，网上很多教程关于此引用的说明是错误的，它们忽略了版本问题。具体可以参考下面这位大佬的：

https://blog.csdn.net/Strong997/article/details/97767929

关于live 2D看板娘的问题

首先我们需要检查博客主目录下的package.json文件里面是否有"hexo-helper-live2d": "^3.0.3",依赖，如果有的话需要先卸载，命令为：npm uninstall hexo-helper-live2d，然后再安装：npm install --save hexo-helper-live2d，此时我们可以在node_modules目录下看到live2d-widget文件，然后我们需要下载看板娘的model，下载好之后可以在node_modules目录下看到形同live2d-widget-model-xxx的文件，这样我们就完成了下载。后续只需要在博客的配置文件里面（如果有主题的话在主题的配置文件里）配置以下代码即可：

live2d:  
  enable: true  
  pluginModelPath: assets/  
  model:  
    use: live2d-widget-model-epsilon2_1  #模板目录，在node_modules里  
  display:  
    position: right  
    width: 150 #人物宽度  
    height: 300 #人物高度  
  mobile:  
    show: false  #是否在手机进行显示
